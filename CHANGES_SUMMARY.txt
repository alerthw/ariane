ARIANE MAP EDITOR - VERSION 1.33 SUMMARY
================================================================================

NEW FEATURES & IMPROVEMENTS (v1.33):
------------------------------------

6. ✅ FEATURE: Russian Localization
   - Full translation of the UI.
   - Toggle language (EN/RU) in the Options menu.

7. ✅ IMPROVEMENT: Manual Transform Input
   - Replaced DragFloat with InputFloat for Translation and Euler XYZ Rotation.
   - Allows precise numeric entry via keyboard.
   - Fixed axes coupling (drift) bug using per-instance caching.

8. ✅ FIX: Keyboard Shortcut Responsiveness
   - Shortcuts (C, B, S, etc.) now work instantly when windows are focused.
   - Blocked only when actively typing in a text field.

9. ✅ FIX: IDE Manifest Injection
   - Ensured 'IDE' directive is written to custom.txt.
   - Prevents game crash (0xC0000005) on custom maps.

------------------------------------

FIXED ISSUES:
-------------

1. ✅ CRITICAL: LOD Index Corruption on Save
   - File: tools/euryopa/fileloader.cpp
   - Issue: LOD indices pointed to wrong objects after save
   - Fix: Recalculate LOD indices based on actual output positions
   - Impact: Game now displays correct LOD models

2. ✅ CRITICAL: Deleted Objects Break Streaming IPL
   - File: tools/euryopa/iplstore.cpp  
   - Issue: Deleted objects caused index misalignment
   - Fix: Reserve nil slots for deleted objects to maintain indices
   - Impact: No more crashes, correct LOD references

3. ✅ HIGH: Silent Save Failures
   - File: tools/euryopa/gui.cpp
   - Issue: Save errors not shown to user
   - Fix: Added Toast notifications for all save errors
   - Impact: Users now see when saves fail

4. ✅ HIGH: Index Desynchronization
   - File: tools/euryopa/fileloader.cpp
   - Issue: m_iplIndex not updated after compacting
   - Fix: Update indices after successful save
   - Impact: Editor and file stay synchronized

5. ✅ MEDIUM: Modloader Documentation
   - File: tools/euryopa/modloader.cpp
   - Issue: Unclear which file types are supported
   - Fix: Added comments and improved documentation
   - Impact: Better understanding of modloader behavior

REMAINING ISSUES (Low Priority):

- Custom IPL always saves to one file (design limitation)
- Theoretical race condition in index updates (rare edge case)
